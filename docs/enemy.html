적 시스템

상속·다형성으로 묶인 적 3종과, 스테이지별로 적을 뽑아 스폰하는 매니저

1-1

Enemy 추상 클래스 — 공통은 부모, 차이는 강제

A/B/C/Boss가 상속하는 부모. 공통 동작은 한 번만, 다른 동작은 abstract로.

Enemy.java
public abstract class Enemy {
    protected int x, y, hp, speed, scoreValue;

    public abstract void move();
    public abstract void update(Player p, List<Bullet> b);
    public abstract void draw(Graphics2D g);
    public abstract Rectangle getBounds();

    public void takeDamage(int d) { hp -= d; }   // 공통 → 부모에 1번
    public boolean isAlive()      { return hp > 0; }
}

핵심: 상속 + 다형성

  • takeDamage·isAlive처럼 모든 적이 똑같이 하는 동작은 부모에 한 번만 구현해 재사용합니다.
  • move/update/draw/getBounds는 적마다 달라서 abstract로 두어 자식이 반드시 구현하게 강제합니다.
  • 덕분에 List<Enemy> 하나에 A·B·C·Boss를 담고 enemy.update()만 호출해도 각자 방식대로 움직입니다 = 다형성.
1-2

EnemyTypeB — sin 곡선으로 흔들리며 하강 ★

y는 일정하게 내려오고, x는 sin 함수로 좌우 진동시키는 계산이 핵심.

EnemyTypeB.java
private static final int    AMPLITUDE  = 40;     // 좌우 폭
private static final double WAVE_SPEED = 0.15;   // 진동 빠르기
private final int    baseX;   // 진동 중심선(생성 위치)
private double       phase;   // 누적 각도(위상)

@Override public void move() {
    y     += speed;            // ① 아래로는 일정하게
    phase += WAVE_SPEED;       // ② 시간 = 각도 누적
    int waveX = baseX + (int) Math.round(
                  AMPLITUDE * Math.sin(phase));   // ③ 좌우 흔들림
    x = Math.max(0, Math.min(waveX,
        GameConstants.SCREEN_WIDTH - GameConstants.ENEMY_WIDTH));
}

계산 원리

매 프레임 phase에 0.15씩 더해 각도를 키우면, Math.sin(phase)가 -1 ~ +1 사이를 주기적으로 오갑니다.

sin(phase) : -1 ───► +1 ───► -1 (반복) x = baseX + 40 × sin(phase) = baseX 기준 좌우 ±40px 왕복
  • ① y += speed : 세로는 직선 하강(EnemyTypeA와 동일).
  • ② phase += 0.15 : "시간"을 각도로 환산. 값이 클수록 더 빨리 흔들립니다.
  • ③ baseX + 진폭×sin : 중심선 baseX 기준으로 좌우 진동. Math.round로 int 좌표에 맞춥니다.
  • 마지막 clamp(Math.max/min)로 화면 밖으로 안 나가게 잡아줍니다.
1-3

EnemyManager — 스폰 간격과 적 종류 추첨

스테이지가 오를수록 더 자주, 더 강한 적이 나오도록 확률을 조절.

EnemyManager.java
private int calculateSpawnInterval(int stage) {
    return Math.max(
        GameConstants.ENEMY_SPAWN_BASE_INTERVAL - stage * 10,  // 점점 빨라짐
        GameConstants.ENEMY_SPAWN_MIN_INTERVAL);               // 하한 25
}

private Enemy selectEnemyType(int roll, int x, int y, int stage) {
    if (stage <= 2) {                       // 초반: A 80% / B 20%
        return roll < 8 ? new EnemyTypeA(x,y) : new EnemyTypeB(x,y);
    } else if (stage < 5) {                 // 중반: A40 B30 C30
        if (roll < 4) return new EnemyTypeA(x,y);
        else if (roll < 7) return new EnemyTypeB(x,y);
        else return new EnemyTypeC(x,y, player);   // 추적형엔 player 주입
    } else {                                // 후반: A30 B30 C40
        if (roll < 3) return new EnemyTypeA(x,y);
        else if (roll < 6) return new EnemyTypeB(x,y);
        else return new EnemyTypeC(x,y, player);
    }
}

난이도 곡선 만들기

  • 스폰 간격: 기본 70프레임에서 스테이지당 10씩 빼되, Math.max(..,25)로 25 밑으로는 안 줄여 난이도 상한을 둡니다.
  • 종류 추첨: 0~9 난수 roll을 구간으로 나눠 비율을 정합니다. 후반일수록 추적형(C) 비율이 30%→40%로 늘어 압박이 커집니다.
  • EnemyTypeC만 생성자에 player를 넘깁니다 — 추적 대상을 알아야 하기 때문.
⚠ 발표 시 인정 포인트: ENEMY_TYPE_B_START_STAGE1 상수는 선언만 되고 안 쓰입니다(정리 대상).
