플레이어 · 총알

이동/클램핑/버프 타이머와 무기 레벨별 탄 발사 벡터

3-1

Player.update — 이동·경계 클램핑·타이머

방향키로 이동하고, 화면 밖 이탈을 막고, 모든 버프 타이머를 1씩 감소.

Player.java
public void update() {
    int speed = speedBuffTimer > 0 ? BASE_SPEED+2 : BASE_SPEED; // 버프 시 +2
    if (left)  x -= speed;   if (right) x += speed;
    if (up)    y -= speed;   if (down)  y += speed;

    x = Math.max(0, Math.min(x, SCREEN_WIDTH  - PLAYER_WIDTH));  // 좌우 가둠
    y = Math.max(0, Math.min(y, SCREEN_HEIGHT - PLAYER_HEIGHT)); // 상하 가둠

    if (invincible) { invincibleTimer--; if (invincibleTimer<=0) invincible=false; }
    if (speedBuffTimer>0) speedBuffTimer--;
    if (shieldTimer>0)  { shieldTimer--;  if (shieldTimer<=0) shieldActive=false; }
    if (attackSpeedBuffTimer>0) attackSpeedBuffTimer--;
}

두 가지 패턴

  • 클램핑: Math.max(0, Math.min(x, 최대))로 좌표를 0~경계 사이에 강제로 가둬 화면 밖으로 못 나가게 합니다.
  • 프레임 타이머: 무적·속도·실드·공속 버프 모두 정수 카운터 하나로 관리. 매 프레임 -- 하다 0이 되면 효과 해제.
  • 좌표계 주의: 자바는 위가 0, 아래로 갈수록 y 증가 → 위로 가려면 y -=.
3-2

Player.shoot — 무기 레벨별 탄 벡터

레벨이 오를수록 중앙 + 좌우로 퍼지는 총알이 누적 발사됨.

Player.java
public void shoot() {
    int bx = x + PLAYER_WIDTH/2 - BULLET_WIDTH/2;   // 정중앙 정렬

    bullets.add(new Bullet(bx, y, 0, -BULLET_SPEED, true));        // 중앙: 위로 직진
    if (weaponLevel >= 2)
        bullets.add(new Bullet(bx+15, y,  2, -BULLET_SPEED, true)); // 우측 퍼짐
    if (weaponLevel >= 3)
        bullets.add(new Bullet(bx-15, y, -2, -BULLET_SPEED, true)); // 좌측 퍼짐
}

속도벡터 (dx, dy)로 방향 표현

중앙 : (dx=0, dy=-5) → 똑바로 위 우측 : (dx=+2, dy=-5) → 위+오른쪽 대각 좌측 : (dx=-2, dy=-5) → 위+왼쪽 대각
  • dy = -BULLET_SPEED : 음수라서 위로 올라갑니다.
  • dx로 좌우 퍼짐을 표현 — 부호와 크기가 곧 방향.
  • if문이 누적이라 레벨3이면 중앙+우+좌 3발이 동시에 나갑니다.
⚠ weaponLevel은 4까지 오르지만 shoot()엔 3까지만 분기 → 레벨4는 레벨3과 동일(미구현).
3-3

Bullet — 직선 등속 이동과 화면 밖 판정

생성 시 정해진 속도벡터로만 직진하고, 화면을 벗어나면 제거 대상이 됨.

Bullet.java
private final int dx, dy;   // final → 생성 후 방향 고정

public void update() {
    x += dx;   // 매 프레임 같은 양만큼
    y += dy;   // 누적 = 등속 직선운동
}

public boolean isOutOfBounds() {
    return y + BULLET_HEIGHT < 0     // 위로 벗어남
        || y > SCREEN_HEIGHT         // 아래로 벗어남
        || x < 0 || x > SCREEN_WIDTH;// 좌우로 벗어남
}

왜 dx, dy가 final인가

  • 생성 시점의 속도벡터로만 움직여 예측 가능한 직선 탄이 됩니다(추적탄도 발사 후엔 직진).
  • isOutOfBounds()가 true면 GamePanel이 리스트에서 제거 → 총알이 무한히 쌓이는 메모리 누수 방지.
  • 플레이어 총알(위로)과 적 총알(아래로)을 한 클래스로 처리하고, playerBullet 플래그로만 구분합니다.
