보스 전투

2페이즈 전환과 부채꼴·추적 탄막의 벡터 계산

2-1

페이즈 전환 — 체력 절반에서 한 번만

HP가 절반 이하로 떨어지면 1→2 페이즈로 단방향 전환.

BossEnemy.java
private void updatePhase() {
    if (hp <= maxHp / 2 && phase == 1) {   // 절반 이하 + 아직 1페이즈
        phase = 2;                         // 두 번 전환 방지(phase==1 조건)
        speed = PHASE2_SPEED;              // 좌우 이동도 빨라짐(2→3)
    }
}

왜 phase == 1 조건이 같이 있나?

  • hp <= maxHp/2는 매 프레임 참이 되므로, 조건이 이것뿐이면 매 프레임 재실행됩니다.
  • && phase == 1을 붙여 딱 한 번만 2페이즈로 넘어가게 합니다(단방향).
  • 한 번에 절반을 넘겨 깎여도 "이하"라서 정상 전환됩니다.
2-2

부채꼴 탄막 — 각도를 등분하는 벡터링 ★

아래 방향을 중심으로 일정 각도 범위를 균등 분할해 여러 발을 퍼뜨림.

BossEnemy.java
private static final double SPREAD_ANGLE = Math.PI / 2.5;  // ≈72°
private static final int    SPREAD_SPEED = 5;

private void fireSpreadBullets(int cx, int cy, int count) {
    for (int i = 0; i < count; i++) {
        double angle = (Math.PI/2)                 // 정아래(90°) 기준
                     - SPREAD_ANGLE/2              // 왼쪽 끝으로
                     + SPREAD_ANGLE * i / Math.max(count-1, 1); // 등분
        int sx = (int)(Math.cos(angle) * SPREAD_SPEED);     // x속도
        int sy = Math.max((int)(Math.sin(angle)*SPREAD_SPEED), 3); // y속도
        bossBullets.add(new Bullet(cx-5, cy, sx, sy, false));
    }
}

각도 → 속도 벡터 변환

각 총알의 발사 각도를 구한 뒤, 삼각함수로 (가로속도 sx, 세로속도 sy) 벡터로 바꿉니다.

중심 = π/2 (정아래) 범위 = SPREAD_ANGLE (72°) i번째 각도 = π/2 − 36° + 72°×(i/(count−1)) sx = cos(angle) × 5 (가로 성분) sy = sin(angle) × 5 (세로 성분)
  • Math.max(count-1, 1) : count=1일 때 0으로 나누기를 막는 안전장치.
  • Math.max(sy, 3) : 거의 수평인 총알도 최소 3px은 내려오게 보장.
  • i=0 → 왼쪽 끝, i=count-1 → 오른쪽 끝 → 부채꼴이 균등하게 펼쳐집니다.
2-3

추적탄 — 플레이어를 향한 단위벡터 ★

발사 순간 플레이어 방향을 구해 크기 1로 정규화한 뒤 속도를 곱함.

BossEnemy.java
private void fireTrackingBullet(int cx, int cy, Player player) {
    int dx = player.getX() + PLAYER_WIDTH/2 - cx;   // x 거리차
    int dy = player.getY() + PLAYER_HEIGHT/2 - cy;  // y 거리차
    double dist = Math.sqrt((double)dx*dx + (double)dy*dy); // 직선거리

    if (dist > 0) {                                  // 0나누기 방지
        int sx = (int)(dx / dist * TRACKING_SPEED);  // 단위벡터 × 속도
        int sy = (int)(dy / dist * TRACKING_SPEED);
        bossBullets.add(new Bullet(cx-5, cy, sx, Math.max(sy,2), false));
    }
}

벡터 정규화(단위벡터)가 핵심

보스→플레이어 방향벡터 (dx, dy)는 길이가 제각각입니다. 이걸 거리(dist)로 나누면 길이 1짜리 방향만 남고, 거기에 속도를 곱해 일정한 빠르기로 날립니다.

dist = √(dx² + dy²) ← 피타고라스 방향(단위벡터) = (dx/dist, dy/dist) ← 길이 1 속도벡터 = 단위벡터 × 6 sx = dx/dist × 6 , sy = dy/dist × 6
  • (double)dx*dx : int 오버플로 방지용 형변환.
  • if (dist > 0) : 플레이어와 겹쳐 거리 0일 때 나누기 오류 방지.
  • 발사 시점 방향으로 고정됩니다(유도 아님) — Bullet의 dx,dy가 final이라서.
