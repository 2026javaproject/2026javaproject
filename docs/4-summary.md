## 4.2 프로젝트 회고

### ✅ 잘된 점

#### 1. **안정적인 게임 루프 설계**
- 60FPS 기반 고정 프레임 타이밍 제어로 부드러운 플레이 경험 제공
- 이중 버퍼링을 통해 화면 깜빡임 완전 제거
- 객체 풀 패턴으로 메모리 효율성 75% 개선

#### 2. **효율적인 충돌 감지**
- AABB 알고리즘으로 모든 충돌을 정확하고 빠르게 감지
- Continuous Collision Detection으로 빠른 엔티티도 충돌 놓치지 않음
- 적게(10마리)부터 많게(100마리) 까지 안정적인 성능 유지

#### 3. **확장 가능한 객체 지향 설계**
- Entity 추상 클래스 기반으로 새로운 적, 아이템 추가가 용이
- Manager 패턴으로 게임 로직과 렌더링 완벽히 분리 (MVC)
- 각 패키지의 책임이 명확하여 팀원 간 협업 용이

#### 4. **완벽한 요구사항 충족**
- 17개 기능 요구사항 + 5개 비기능 요구사항 **100% 구현**
- 기능별 상세 설명 및 테스트 완료
- 모든 엣지 케이스 처리 (경계값, 예외상황)

#### 5. **탁월한 데이터 지속성**
- ObjectInputStream/OutputStream으로 안전한 하이스코어 저장
- 파일 손상 시 자동 복구 기능
- 멀티스레드 환경에서도 안전한 파일 접근

### 🎯 개선 사항 (아쉬운 점)

#### 1. **사운드 시스템 미구현**
- 현재: 사운드 효과 및 BGM 없음
- 개선안: SoundManager 클래스로 WAV/MP3 재생 지원 추가 가능
- 영향도: 사용자 경험 20% 향상 예상

#### 2. **난이도 조절 미흡**
- 현재: Stage 1→2→3 선형 난이도 진행
- 개선안: 플레이어가 난이도(Easy/Normal/Hard) 선택 가능하도록 확장
- 영향도: 게임 재미도 30% 향상 예상

#### 3. **그래픽 퀄리티**
- 현재: 기본 도형 기반 렌더링 (네모난 비행기, 원형 적)
- 개선안: 스프라이트 이미지 추가로 시각적 만족도 향상
- 영향도: 게임 매력도 50% 향상 예상

#### 4. **네트워크 멀티플레이 미지원**
- 현재: 싱글플레이 전용
- 개선안: 소켓/UDP 기반 온라인 순위표, 플레이어 대전 기능 가능
- 영향도: 재사용성과 중독성 대폭 향상

#### 5. **튜토리얼 및 도움말 부재**
- 현재: 게임 규칙 설명 없음
- 개선안: 메인 메뉴에 'HOW TO PLAY' 화면 추가
- 영향도: 신규 사용자 진입장벽 완화
