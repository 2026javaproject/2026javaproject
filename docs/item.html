5-1

enum + switch 표현식 — 종류별 효과

아이템 종류를 enum으로 두고 화살표 switch로 안전하게 분기.

PowerUpItem.java
public enum ItemType { SPEED_UP, MULTI_SHOT, SHIELD, ATTACK_SPEED }

public void applyEffect(Player p) {
    switch (type) {
        case SPEED_UP     -> p.addSpeedBuff(180);                  // 3초
        case MULTI_SHOT   -> p.upgradeWeapon(p.getWeaponLevel()+1);// 영구 +1
        case SHIELD       -> p.addShield(SHIELD_DURATION);         // 4초
        case ATTACK_SPEED -> p.addAttackSpeedBuff(240);            // 4초
    }
    collected = true;
}

enum을 쓰는 이유

  • 모든 case를 처리하면 컴파일러가 default 생략을 허용합니다(누락 시 경고).
  • 화살표(->) switch는 break가 필요 없고 fall-through 위험이 없습니다.
  • MULTI_SHOT만 영구(무기 레벨업), 나머지 3개는 프레임 타이머 기반 일시 버프.
5-2

sin 펄스 — 반짝이는 빛 효과 계산 ★

시간에 따라 0~1을 오가는 값을 만들어 크기·투명도를 주기적으로 변화.

PowerUpItem.java
public void draw(Graphics2D g) {
    // pulseTimer는 매 프레임 ++
    double pulse = (Math.sin(pulseTimer * 0.18) + 1.0) / 2.0; // 0~1
    int glowSize = SIZE + (int)(10 * pulse);          // 23~33px
    int glowX = x - (glowSize - SIZE)/2;              // 중심 유지 보정
    int glowY = y - (glowSize - SIZE)/2;
    int alpha = 60 + (int)(120 * pulse);              // 60~180 투명도
    g.setColor(new Color(c.getRed(),c.getGreen(),c.getBlue(), alpha));
    g.fillOval(glowX, glowY, glowSize, glowSize);     // 외부 빛
    ...
}

sin을 0~1로 바꾸는 정석 공식

sin(t) 범위 : −1 ~ +1 (sin(t)+1)/2 → 0 ~ 1 로 변환 glow 크기 = 23 + 10×pulse (23~33) 투명도 alpha = 60 + 120×pulse (60~180)
  • pulseTimer*0.18 : 시간을 각도로 환산(클수록 빨리 깜빡).
  • (sin+1)/2 : -1~1을 0~1로 옮기는 흔한 트릭(크기·투명도 등 양수 값에 매핑).
  • 커진 만큼의 절반을 좌표에서 빼서((glowSize-SIZE)/2) 빛이 중심을 기준으로 확대되게 합니다.
5-3

ItemManager — 확률 드롭과 보스 분산 드롭

일반 적은 25% 확률, 보스는 확정 2개를 주변에 흩뿌림.

ItemManager.java
public void tryDrop(int ex, int ey, List<PowerUpItem> items) {
    if (random.nextInt(DROP_CHANCE) == 0) {       // 1/4 = 25%
        ItemType[] types = ItemType.values();
        ItemType t = types[random.nextInt(types.length)];
        items.add(new PowerUpItem(t, ex, ey));
    }
}

public void dropBossItems(int bx, int by, List<PowerUpItem> items) {
    for (int i = 0; i < BOSS_DROP_COUNT; i++) {   // 확정 2개
        int offX = random.nextInt(60) - 30;       // -30~+29
        int offY = random.nextInt(40) - 20;       // -20~+19
        ItemType t = ItemType.values()[random.nextInt(4)];
        items.add(new PowerUpItem(t, bx+offX, by+offY)); // 주변 분산
    }
}

확률과 분산 좌표

  • nextInt(4)==0 : 0~3 중 0일 확률 = 정확히 25%.
  • 오프셋 트릭: nextInt(60)-30 → 0~59에서 30을 빼 -30~+29 범위를 만들어 보스 좌우로 고르게 흩뿌립니다.
  • ItemType.values()로 종류를 뽑아, 아이템이 추가돼도 이 코드는 그대로 동작합니다.
