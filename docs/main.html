메인 · 화면전환

JFrame 화면 전환과 EDT 진입점

8-1

화면 전환 — setContentPane 한 줄로 교체

메뉴 → 게임 → 게임오버를 패널 교체로 전환.

GameMain.java
public class GameMain extends JFrame {
    public void startGame() {
        gamePanel = new GamePanel(this);
        setContentPane(gamePanel);   // 콘텐츠 영역 통째 교체
        revalidate();                // 레이아웃 재계산
        gamePanel.startGameLoop();   // 게임 루프 시작
    }
    public void showGameOver(int score) {
        setContentPane(new GameOverPanel(this, score));
        revalidate();
    }
}

JFrame 상속 구조

  • extends JFrame → this가 곧 창. setContentPane() 한 줄로 화면 전체 내용을 바꿉니다.
  • 각 패널 생성 시 this(GameMain)를 넘겨, 패널이 다시 GameMain의 화면전환 메서드를 호출할 수 있게 합니다.
  • revalidate()로 새 패널 레이아웃을 다시 계산.
8-2

main — EDT에서 GUI 생성

Swing 규칙: GUI는 이벤트 디스패치 스레드에서 만들어야 안전.

GameMain.java
public static void main(String[] args) {
    SwingUtilities.invokeLater(GameMain::new);
}

왜 invokeLater인가

  • Swing 컴포넌트는 반드시 EDT(Event Dispatch Thread)에서 생성·수정해야 합니다.
  • main 스레드에서 바로 만들면 렌더링 깨짐·경쟁 상태 같은 비결정적 버그 위험.
  • GameMain::new : 메서드 참조로 생성자를 EDT 작업 큐에 등록.
8-3

생성자 — 창 기본 설정

제목·크기·종료동작·위치를 정하고 첫 화면을 띄움.

GameMain.java
public GameMain() {
    setTitle("PILOT");
    setSize(SCREEN_WIDTH, SCREEN_HEIGHT);          // 480×720
    setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);// X → 프로세스 종료
    setResizable(false);                           // 크기 고정
    setLocationRelativeTo(null);                   // 화면 중앙
    showMainMenu();                                // 첫 화면
    setVisible(true);
}

설정 포인트

  • setResizable(false) : 모든 좌표가 480×720을 가정하므로 크기를 고정합니다.
  • setLocationRelativeTo(null) : 창을 모니터 중앙에 배치.
  • 크기를 상수로 받아 다른 클래스(좌표 계산)와 항상 일치시킵니다.
⚠ setSize는 타이틀바 포함 크기 → 실제 게임 영역은 조금 작아질 수 있음(getContentPane 기준 권장).
