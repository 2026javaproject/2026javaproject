# 9. AI 개발 지원 프롬프트 모음

## 9.1 사용 목적

이 문서는 프로젝트 개발 과정에서 **GitHub Copilot CLI**를 활용한 프롬프트 모음입니다.
- 코드 생성
- 문서화
- 성능 최적화
- 버그 수정

## 9.2 초기 설계 단계

### 프롬프트 1: 프로젝트 구조 설계
```
"Java Swing 기반 2D 슈팅 게임 PILOT의 패키지 구조를 설계해줘.
6개 패키지(main, entity, manager, screen, ui, util)로 나누고,
각 패키지의 책임과 클래스를 정의해줘."
```

### 프롬프트 2: 요구사항 분석
```
"게임의 17개 기능 요구사항(F-01~F-17)을 마크다운 테이블로 정리해줘.
각 요구사항에 ID, 기능명, 개요, 우선순위, 담당자를 포함해줘."
```

## 9.3 구현 단계

### 프롬프트 3: 게임 루프 구현
```
"Java Swing에서 60FPS 게임 루프를 구현해줘.
- update() 메서드
- paintComponent() 렌더링
- 프레임 타이밍 제어 (16.67ms)
- 이중 버퍼링 적용"
```

### 프롬프트 4: Player 클래스
```
"Player 클래스를 구현해줘.
- 방향키/WASD 이동 (속도 5)
- Space 자동 발사 (12프레임 쿨다운)
- 무적 시간 (90프레임)
- 화면 경계 처리"
```

### 프롬프트 5: AABB 충돌 감지
```
"AABB 충돌 감지를 구현해줘.
- Rectangle.intersects() 사용
- 플레이어 총알 ↔ 적
- 적 총알 ↔ 플레이어
- 아이템 ↔ 플레이어"
```

### 프롬프트 6: Enemy 계층 구조
```
"적 클래스 계층을 설계해줘.
- Enemy (추상 클래스)
- EnemyTypeA (직선 이동)
- EnemyTypeB (사인파 이동)
- EnemyTypeC (플레이어 추적)
- BossEnemy (2페이즈)"
```

### 프롬프트 7: 점수 시스템
```
"ScoreManager 싱글톤으로 점수 관리해줘.
- 적 종류별 차등 점수
- 스테이지 진행 판정
- 하이스코어 저장/불러오기
- 게임 상태 관리"
```
