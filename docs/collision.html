충돌 시스템

AABB 충돌과 역순 순회, 다형성 데미지 적용

4-1

총알 ↔ 적 충돌 — AABB + 역순 순회

사각형 겹침으로 빠르게 판정하고, 리스트를 뒤에서부터 돌며 안전하게 제거.

CollisionDetector.java
public static void checkBulletEnemyCollision(List<Bullet> bullets, List<Enemy> enemies) {
    for (int i = bullets.size()-1; i >= 0; i--) {     // 뒤에서부터
        Bullet b = bullets.get(i);
        if (!b.isPlayerBullet()) continue;            // 적 총알은 스킵
        for (int j = enemies.size()-1; j >= 0; j--) {
            Enemy e = enemies.get(j);
            if (b.getBounds().intersects(e.getBounds())) {  // AABB
                applyBulletDamage(b, e);
                bullets.remove(i);
                break;                                 // 1발 = 1적
            }
        }
    }
}

두 기법

  • AABB(축정렬 사각형): Rectangle.intersects()는 x/y/폭/높이만 비교 → 매우 빠름. 슈팅게임엔 충분.
  • 역순 순회: remove(i) 시 뒤 요소가 앞으로 당겨지는데, 뒤에서부터 돌면 이미 처리한 인덱스라 인덱스가 안 꼬입니다.
  • break : 한 발이 여러 적을 관통하지 못하게 막음(밸런스).
4-2

데미지 적용 — 다형성으로 보스까지 한 번에

보스도 Enemy를 상속하므로 같은 메서드로 데미지를 줄 수 있음.

CollisionDetector.java
private static void applyBulletDamage(Bullet bullet, Enemy enemy) {
    int damage = (int) Math.ceil(bullet.getDamage()); // float→올림 정수
    enemy.takeDamage(damage);   // Enemy 타입이면 누구든 OK
}

// 보스 충돌도 같은 메서드 재사용
if (bullet.getBounds().intersects(boss.getBounds())) {
    applyBulletDamage(bullet, boss);   // boss는 BossEnemy(=Enemy)
    bullets.remove(i);
}

다형성의 이점

  • 매개변수 타입이 Enemy라, EnemyTypeA/B/C든 BossEnemy든 구분 없이 같은 코드로 처리합니다.
  • Math.ceil : 데미지가 1.5 같은 float여도 올림해 최소 1 이상 정수 데미지를 보장.
  • 일반 적과 보스에 중복 코드를 안 쓰고 한 메서드로 통일 → 유지보수 쉬움.
4-3

플레이어 ↔ 아이템 — break 없는 이유

한 프레임에 여러 아이템을 동시에 먹을 수 있으므로 끝까지 검사.

CollisionDetector.java
public static void checkPlayerItemCollision(Player player, List<PowerUpItem> items) {
    Rectangle pb = player.getBounds();
    for (int i = items.size()-1; i >= 0; i--) {
        PowerUpItem item = items.get(i);
        if (pb.intersects(item.getBounds())) {
            item.applyEffect(player);   // 종류별 버프 적용
            items.remove(i);            // ★ break 없음
        }
    }
}

충돌마다 break 여부가 다르다

  • 적·총알 피격은 한 번만 처리하면 되므로 break를 씁니다.
  • 아이템은 여러 개를 동시에 획득 가능해서 break 없이 끝까지 돌며 전부 먹습니다.
  • player.getBounds()는 반복 밖에서 한 번만 호출해 재사용.
