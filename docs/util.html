유틸 · 저장

프레임 기반 상수와 방어적 파일 입출력

7-1

GameConstants — 모든 시간은 프레임 단위

매직넘버를 없애고, 시간을 60FPS 기준 프레임 수로 통일.

GameConstants.java
public static final int FPS   = 60;
public static final int DELAY = 1000 / FPS;   // ≈16ms (한 프레임 대기)

public static final int PLAYER_INVINCIBLE_DURATION = 90;  // 1.5초
public static final int SHIELD_DURATION            = 240; // 4초
public static final int ENEMY_SPAWN_MIN_INTERVAL   = 25;  // 스폰 하한

프레임 기반 설계

  • 60FPS 가정 하에 90프레임=1.5초, 240프레임=4초. 타이머를 정수 카운터로 단순화.
  • 모든 필드가 public static final → 인스턴스 불필요, 한 곳만 고치면 전체 반영.
⚠ DELAY=1000/FPS는 정수 나눗셈이라 16(16.67 손실) — FPS를 바꾸면 미세 오차 가능.
7-2

loadHighScore — 방어적 파싱

파일이 없거나 깨져도 절대 게임이 멈추지 않게 0으로 안전 처리.

HighScoreFileIO.java
public static int loadHighScore() {
    Path path = Path.of(HIGH_SCORE_FILE);
    if (!Files.exists(path)) return 0;             // 파일 없음
    try {
        String content = Files.readString(path, UTF_8).trim();
        if (content.isEmpty()) return 0;
        String cleaned = content.replaceAll("[^0-9-]", ""); // 숫자/− 만
        if (cleaned.isEmpty()) return 0;
        return Integer.parseInt(cleaned);
    } catch (IOException | NumberFormatException e) {
        return 0;                                   // 어떤 오류든 0
    }
}

방어적 프로그래밍

  • 파일이 없음 / 비었음 / 손상됨 / 숫자 아님 — 모든 경우 0 반환해 게임이 죽지 않게 합니다.
  • replaceAll("[^0-9-]","") : 사용자가 손으로 수정하거나 공백·BOM이 섞여도 숫자만 추출.
  • 두 예외를 |로 한 번에 처리(멀티캐치).
7-3

saveIfHighScore — 갱신일 때만 쓰기

기존 기록보다 높을 때만 파일을 덮어쓰고, 저장 실패도 무시.

HighScoreFileIO.java
public static int saveIfHighScore(int score) {
    int best = loadHighScore();
    if (score > best) { writeHighScore(score); return score; }
    return best;
}

private static void writeHighScore(int score) {
    try {
        Files.writeString(Path.of(HIGH_SCORE_FILE),
                          String.valueOf(score), UTF_8);
    } catch (IOException e) { /* 무시: 게임 진행 우선 */ }
}

설계 의도

  • 비교 → 갱신 시에만 쓰기 → 새 최고점수 반환을 한 메서드로 묶어 호출부(게임오버 화면)가 단순해집니다.
  • 저장 실패(권한·디스크 문제)도 예외를 삼켜 게임 진행을 우선합니다.
