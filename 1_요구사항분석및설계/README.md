## 1-2. 요구사항 상세 내용

#### F-01 — 메인 메뉴 UI
| 항목 | 내용 |
|------|------|
| 입력 | 마우스 클릭 또는 키보드 Enter / Space |
| 처리 | Rectangle.contains() 로 버튼 클릭 영역 감지 |
| 출력 | START GAME / HIGH SCORE / EXIT 버튼 표시 |
| 예외 | 최고점수 파일 없을 시 BEST: 0 으로 표시 |

#### F-02 — 플레이어 이동
| 항목 | 내용 |
|------|------|
| 입력 | 방향키(↑↓←→) 또는 WASD |
| 처리 | boolean 플래그를 매 프레임 확인해 좌표 변경 |
| 출력 | 화면 경계 안에서 부드럽게 이동 |
| 예외 | 화면 끝 도달 시 해당 방향 이동 불가 (클램핑) |

#### F-03 — 총알 발사
| 항목 | 내용 |
|------|------|
| 입력 | 자동 (별도 키 불필요) |
| 처리 | shootTimer 가 12프레임 도달마다 Bullet 생성 |
| 출력 | 초당 5발, 플레이어 중앙 상단 방향으로 발사 |
| 예외 | 일시정지 상태에서는 발사 중단 |

#### F-05 — 적 기체 스폰 및 이동
| 항목 | 내용 |
|------|------|
| 입력 | 프레임 카운터, 현재 스테이지 번호 |
| 처리 | frameCounter % interval == 0 인 프레임마다 스폰 |
| 출력 | 화면 상단 랜덤 위치에서 적 생성 |
| 예외 | 보스 전투 중에는 일반 적 스폰 중단 |

| 스테이지 | TypeA 직선 | TypeB 사인파 | TypeC 추적 |
|----------|-----------|-------------|-----------|
| 1 | 80% | 20% | 0% |
| 2 | 40% | 30% | 30% |
| 3 이상 | 20% | 40% | 40% |

#### F-06 — 보스 등장 및 공격 패턴
| 항목 | 내용 |
|------|------|
| 등장 조건 | 스테이지별 처치 수 임계값 도달 (1단계:15, 2단계:25, 3단계:40) |
| 페이즈 1 | HP 30, 70프레임마다 3발 부채꼴 발사 |
| 페이즈 2 | HP 15 이하 진입, 40프레임마다 5발 + 조준 발사 |
| 처치 보상 | 1,000점 + 파워업 아이템 3종 전체 드롭 |

#### F-11 — 아이템 드롭 및 효과
| 아이템 | 효과 | 지속 |
|--------|------|------|
| SPEED UP | 이동속도 +2 | 180프레임 (3초) |
| MULTI SHOT | 3방향 동시 발사 활성화 | 스테이지 종료까지 |
| SHIELD | 다음 피격 1회 완전 무효화 | 1회 |

### 2. 비기능적 요구사항 정리 (Non-Functional)

| ID | 요구사항 | 내용 |
|:---|:---------|:-----|
| NF-01 | 성능 (FPS) | 실시간 게임 루프를 통해 끊김 없는 렌더링(최소 60FPS)을 보장해야 한다 |

## 조작 및 시스템 동작 원리 (Controls & System Mechanics)

본 프로젝트는 사용자의 키보드 입력이 내부 로직을 거쳐 화면에 렌더링 되기까지 다음과 같은 체계적인 흐름을 통해 구동됩니다.

| 시스템 구분 | 핵심 기술 및 요소 | 동작 및 처리 과정 설명 |
| --- | --- | --- |
| **사용자 조작** | `KeyListener` | 사용자는 키보드 방향키를 통해 전투기를 조작하고, Enter 키로 메뉴를 선택합니다. 화면과 UI 단에서는 `KeyListener`를 통해 이러한 사용자 입력을 실시간으로 감지하고, 그 신호를 즉시 메인 컨트롤러와 Manager 객체로 전달하여 상태 변화를 유도합니다. |
| **화면 렌더링** | `Double Buffering` | UI는 내부 상태값(좌표, 체력 등)을 바탕으로 화면을 그려냅니다. 이때 그림이 그려지는 과정이 사용자 눈에 보여 화면이 깜빡이는 현상(Flickering)을 방지하기 위해 **더블 버퍼링(Double Buffering)** 기법을 활용합니다. 보이지 않는 가상 메모리 공간(버퍼)에 다음 프레임의 그림을 미리 모두 그려둔 뒤, 완성된 이미지를 한 번에 모니터로 덮어씌워 화면을 매우 부드럽게 갱신합니다. |
| **게임 루프** | `Thread` | 메인 클래스는 자바의 별도 스레드(Thread)를 생성하여 백그라운드에서 **게임 루프(Game Loop)**를 쉼 없이 가동합니다. 이 루프는 정해진 주기(예: 1초에 60번)마다 지속적으로 로직 업데이트 메서드와 화면 갱신 메서드를 호출하여, 게임 내의 시간이 멈추지 않고 일정하게 흐르도록 만듭니다. |
| **로직 제어** | `Manager` | 메인 게임 루프가 돌아가는 동안, 실질적인 물리 법칙과 게임 규칙은 Manager 패키지가 전담합니다. 사용자 입력과 시간에 따라 객체(엔티티)들의 이동 좌표를 갱신하고, 총알과 적기의 충돌을 감지하며, 정해진 확률에 따라 적과 아이템을 스폰(생성)하고, 처치 시 점수를 계산하는 등 모든 백엔드 연산을 처리합니다. |
| **데이터 보존** | `util` (File I/O) | 게임 플레이가 종료(Game Over)되면 `util` 패키지가 활성화되어 현재 점수와 기존 최고 점수를 비교합니다. 갱신이 필요할 경우 파일 입출력(File I/O) 기능을 통해 로컬 파일에 최종 점수를 안전하게 저장합니다. 이후 프로그램을 다시 실행하여 메인 메뉴를 로드할 때, 이 로컬 파일을 읽어와 시스템에 최고 점수를 반영 및 출력합니다. |

#### Player (플레이어 전투기)

| 필드명 | 타입 | 설명 | 초기값 |

| x | int | 스프라이트 좌상단 X 좌표 | 화면 중앙 |

| shootTimer | int | 자동 발사 쿨다운 카운터 | 0 |
| invincibleTimer | int | 무적 남은 프레임 수 | 0 |
| speedBoostTimer | int | 속도 증가 남은 프레임 수 | 0 |
| shielded | boolean | 실드 활성화 여부 | false |
