## >> 시스템 설계

### 전체 프로젝트 개요 다이어그램

```
[사용자 입력 (키보드)] ──> [화면/UI (Java Swing/AWT)] ──> [메인 (Game Loop)] ──> [manager / 엔티티 (Business Logic)] ──> [util (File I/O)]
```

- 사용자는 키보드를 통해 기체를 조작하고 메뉴를 선택한다
- 화면/UI는 KeyListener를 통해 입력을 받아 컨트롤러(메인/manager)로 전달하고, 상태를 그린다(Double Buffering 활용)
- 메인은 스레드를 이용한 게임 루프(Game Loop)를 돌며 주기적으로 화면 갱신과 로직 업데이트를 호출한다
- manager는 엔티티의 이동, 충돌 감지, 스폰, 점수 계산 등을 처리한다
- util은 게임 종료 시 최종 점수를 로컬 파일에 저장하거나, 메인 메뉴 로드 시 최고 점수를 읽어온다

### 주요 클래스 데이터 구조 (필드 정리)

DB를 사용하지 않으므로, 핵심 엔티티 및 파일 I/O 데이터 구조를 정의합니다.

| 클래스 / 파일 | 주요 필드 (속성) |
|:---|:---|
| Entity (추상) | x, y, width, height, speed, image |
| Player | hp, maxHp, score, bulletType, isDead |
| Enemy / Boss | hp, scoreReward, patternState |
| Score.dat (파일) | highest_score (정수 또는 암호화된 문자열) |

### 화면 설계 (Wireframe)

#### 메인 화면 (화면/MainMenu)
- 화면 중앙: 크게 렌더링된 "PILOT" 타이틀 로고
- 화면 하단: 'Game Start', 'Exit' 등 키보드/마우스로 선택 가능한 메뉴 버튼
- 화면 상단 모서리: util을 통해 불러온 High Score 표시

#### 인게임 화면 (화면/GameScreen & UI/HUD)
- 배경: 횡스크롤 느낌을 주는 흐르는 우주/하늘 배경 애니메이션
- 메인 영역: 플레이어 기체(좌측 위치), 우측에서 좌측으로 이동하는 적 기체 및 장애물 렌더링
- 상단 HUD: 현재 점수, 플레이어 HP 바(게이지 형태), 현재 스테이지 및 보스 출현 게이지 표시

#### 게임 오버 화면 (화면/GameOver)
- 화면 중앙: "GAME OVER" 텍스트와 함께 이번 판에 획득한 최종 점수 표시
- 조건부 텍스트: 기존 최고 점수를 넘었을 경우 "NEW HIGH SCORE!" 애니메이션 출력
- 하단 메뉴: '다시 하기(Restart)', '메인 메뉴로 돌아가기' 안내 문구
